# 第六阶段：多账户

状态：已验收。

日期：2026-07-17

## 一、阶段目标

在现有单账户 IMAP、本地优先阅读、完整同步语义和 SMTP 发件基础上，支持多个相互隔离的密码账户，并使账户新增、切换、编辑连接、重新认证和移除都能在应用运行期间完成。

- 每个账户拥有独立的匿名数据槽、系统凭据、同步策略、文件夹映射、同步状态和后台 Supervisor。
- 应用启动后为所有已配置账户恢复后台同步；切换账户只切换本地数据视图，不等待网络连接。
- 主界面继续以账户为第一层，不增加统一收件箱，也不跨账户合并文件夹、搜索结果或未读计数。
- 账户设置在独立设置窗口内完成；新增账户复用首次启动时已经验证的发现、手动配置和 IMAP/SMTP 双认证流程。
- 本阶段只支持 IMAP + SMTP 密码账户。POP3、Google/Microsoft OAuth、身份/别名和跨账户统一能力仍留在后续阶段。

## 二、阶段决策

### 账户身份与重复判断

- `account_id` 是本机账户配置的稳定 ID，`data_slot_id` 是可迁移数据目录中的匿名隔离 ID，两者不互相替代。
- 同一邮箱地址可以用于不同服务器或用户名，但完全相同的邮箱地址、IMAP 主机、端口和登录用户名组合不能重复添加。
- 账户顺序按创建时间保持稳定；本阶段不实现拖动排序。
- 最近选择的账户 ID 保存在系统应用配置区。启动时优先恢复；不存在时回退到第一个账户。

### 移除账户的数据语义

- “移除账户”停止该账户的后台任务，关闭该账户的写信窗口，删除本机账户配置并清理系统凭据。
- 可迁移数据目录中的匿名账户槽、邮件、附件和草稿默认保留，不在本阶段执行破坏性删除；后续迁移与硬化阶段再提供孤立数据槽识别、重绑定和显式清理。
- 正在发送的任务或仍需写回服务器的待办操作会阻止移除，并显示可理解的原因；仅有普通本地草稿不会阻止移除。
- 凭据清理失败不恢复已经移除的账户；只保留不含邮箱地址或秘密的凭据引用清理任务，并在后续启动重试。

### 连接和认证更新

- 编辑连接时预填邮箱、显示名称和服务器配置，永不回填密码。
- 服务器配置或密码变更必须重新通过 IMAP 登录/Capability 和 SMTP 登录测试，两端成功后才原子替换配置与系统凭据。
- 只有凭据缺失或服务端明确返回认证失败时进入 `reauth_required`；网络错误、TLS 错误和服务器临时错误不能误判为需要重新认证。
- 进入 `reauth_required` 后停止该账户的自动登录重试，保留本地邮件可读，并在用户完成重新认证后立即恢复 Supervisor。

## 三、Rust 运行模型

### Supervisor Registry

将当前单一 `MailRuntime` 循环重构为：

- `SupervisorRegistry`：读取账户配置并维护 `account_id -> AccountSupervisor`；处理启动、新增、更新、移除和关闭。
- `AccountSupervisor`：拥有账户独立的唤醒信号、手动/后台同步请求、退避、启动同步状态、待办重放和 IDLE 生命周期。
- 共享 `MailRepository`：继续通过 `account_slot_id` 隔离所有查询和写入；不为每个账户重复打开 SQLite 连接池。
- 全局网络信号量：最多同时执行两个主动 IMAP 同步/写操作；同一账户仍保持串行。IDLE 等待不占用主动同步名额。

每个账户最多保持一个 Inbox IDLE 会话和一个按需命令会话。手动收取当前账户拥有高优先级，但后台任务使用轮转调度，避免长期饿死其他账户。现有 5 分钟全文件夹对账、无 IDLE 时 60 秒轮询和 2 秒至 5 分钟退避继续按账户独立运行。

### 账户变更协调

1. 新增账户事务成功后发布账户修订事件，Registry 创建 Supervisor 并立即开始首次同步，无需重启应用。
2. 更新连接或重新认证前暂停该账户新任务；验证成功后替换配置/凭据并重启该账户连接，其他账户不受影响。
3. 移除前将账户置为 `removing`，拒绝新的写操作，立即取消 IDLE/退避等待，并对正在执行的协议步骤做最长 10 秒的有界收尾；该账户写信窗口先保存未提交内容再关闭。
4. 账户配置移除后注销 Supervisor；该账户迟到事件通过账户代次丢弃，不能重新写入界面状态。

### 发件并发

- `SendWorker` 从当前全局串行扫描改为账户内 FIFO、账户间公平调度。
- 全局最多同时发送两封邮件，同一账户一次只发送一封，避免单个 SMTP 超时阻塞其他账户。
- 每个任务继续使用创建时的 `account_slot_id` 和不可变 MIME；发送前必须解析到仍存在的账户配置与凭据。
- 认证失败只影响对应账户和任务，不暂停其他账户的发送与同步。

## 四、配置、凭据与数据隔离

- `accounts.json` 从单账户假设升级为可并发安全更新的多账户集合，并增加单调账户修订号及待清理凭据引用。
- 所有账户配置变更使用进程内串行写锁和原子文件替换，避免两个设置操作互相覆盖。
- 每个密码账户继续使用独立 `credential_ref`，密码只写入 Windows Credential Manager/macOS Keychain。
- 新增账户沿用“验证连接、创建匿名数据槽、写入凭据、保存账户配置”的补偿事务；任何一步失败不保留可见半成品账户。
- 更新账户先用旧凭据或用户提供的新密码验证候选配置。需要替换密码时写入新的临时凭据引用，再原子切换账户配置，最后清理旧凭据；任一步失败时旧配置、旧凭据和旧 Supervisor 继续可用。
- 移除账户先提交配置移除，再清理凭据；清理失败进入幂等重试队列，不记录密码、Token 或服务器响应。
- 审计所有 Repository 接口：即使前端传入另一账户的消息、文件夹、附件、草稿或任务 ID，也必须因 `account_slot_id` 不匹配而返回 `not_found`。

现有 `account_slots` 以及邮件、文件夹、草稿、发件任务和同步设置的 `account_slot_id` 外键已经支持多账户，本阶段原则上不改变邮件主体模型；只在确有需要时增加配置修订或运行状态迁移。

## 五、公共类型、命令与事件

新增或调整类型：

- `AccountRuntimeState = starting | ready | syncing | offline | retrying | reauth_required | removing | stopped`
- `AccountRuntimeSummary { account_id, state, error_code?, retry_at?, revision }`
- `AccountConnectionDraft`：可编辑账户配置，不包含密码。
- `AccountRemovalImpact { editing_drafts, queued_send_jobs, pending_operations, can_remove }`
- 本机偏好增加 `last_selected_account_id?`。

新增或调整命令：

- `add_password_account(draft)`：取代仅限首次启动的单账户保存限制，首次启动和设置页共同复用。
- `get_account_connection_draft(account_id)`
- `update_password_account(account_id, draft, new_password?)`：未提供新密码时由 Rust 使用现有系统凭据验证，密码不会返回前端。
- `reauthenticate_password_account(account_id, password)`
- `get_account_removal_impact(account_id)`
- `remove_account(account_id)`
- `list_account_runtime_summaries()`
- `set_last_selected_account(account_id)`

现有邮件、草稿和发件命令继续显式接收 `account_id`。事件只携带稳定 ID、状态和修订号：

- `accounts-changed { revision }`
- `account-runtime-status-changed { account_id, state, revision }`
- `account-removing { account_id }`

前端收到事件后失效对应查询，不通过事件推送账户服务器配置、凭据或完整邮件数据。

## 六、前端交互

### 主窗口

- 只有一个账户时继续直接显示账户身份；两个及以上账户时显示现有下拉切换器。
- 切换账户立即重置文件夹、邮件选择和当前搜索，读取目标账户 SQLite 视图，并持久化最近选择。
- 非当前账户的后台事件只失效该账户缓存，不打断当前阅读、不显示同步进度条。
- 下拉菜单显示各账户名称、邮箱和简洁运行状态；认证失效账户提供“需要重新认证”提示，但本地邮件仍可切换阅读。
- 不增加统一收件箱、跨账户搜索或跨账户批量操作。

### 设置窗口

- “账户”类别先显示账户列表和“添加账户”主按钮；选择账户后显示连接状态、同步策略、文件夹映射、编辑连接、重新认证和移除操作。
- 添加流程复用账户发现与手动配置表单，但不出现首次启动步骤或数据目录内容。
- 编辑和重新认证使用独立表单状态；关闭设置窗口不会保存未提交内容。
- 移除前展示影响摘要和二次确认，明确说明本地邮件数据仍保留；有待发送任务或待办操作时解释阻止原因。
- 添加、更新、移除成功后主窗口账户切换器即时更新，无需刷新或重启。

## 七、实施顺序

1. 移除 `account.first_iteration_limit`，为账户配置集合增加修订、串行原子更新、重复检测和凭据清理队列测试。
2. 抽取首次启动账户表单，使 onboarding 和设置页共享发现、手动配置、连接测试与安全警告逻辑。
3. 实现添加、编辑连接、重新认证、移除影响检查和账户移除业务用例及补偿逻辑。
4. 将单一 MailRuntime 循环拆为 Supervisor Registry 与账户级 Supervisor，接入全局主动网络信号量和账户生命周期协调。
5. 调整 SendWorker 为账户间公平、账户内 FIFO 的有限并发模型。
6. 审计并补强所有 Repository 和 Command 的 `account_id -> data_slot_id` 隔离校验。
7. 改造主窗口账户查询、切换持久化、事件失效和认证失效状态；改造设置窗口账户列表与管理页面。
8. 补齐中英文文案、正式测试、架构文档和变更记录，完成自动验证后交由用户手动验收。

## 八、自动验证

- AppService：多个账户添加、重复拒绝、配置原子更新、凭据写入失败补偿、移除与凭据清理重试。
- Registry：启动恢复、运行期新增、配置更新重启、移除注销、重复事件和应用关闭清理。
- 并发：主动 IMAP 操作不超过两个、同账户不并行、手动同步优先且后台账户不饿死。
- 认证状态：凭据缺失/明确认证失败进入 `reauth_required`，网络/TLS/临时服务器错误保持重试语义。
- 发件：账户内 FIFO、账户间公平、一个账户超时或认证失败不阻塞另一个账户。
- 数据隔离：跨账户消息、附件、文件夹、草稿、发件任务和待办 ID 均无法越权读取或修改。
- 前端：单账户不显示切换下拉、多账户可以切换、切换清理选择、账户事件刷新、设置页增删改与二次确认。
- 执行前端测试、TypeScript/Vite 构建、Rust 单元/集成测试和 Clippy；按约定不执行完整 Tauri bundle。

协议测试继续使用测试 Adapter 或进程内服务，不固化真实账户。临时目录、凭据和探针在测试结束后清理，正常构建缓存保留。

### 实施结果

- `accounts.json` 已兼容旧格式并增加修订号、最近选择账户和凭据清理队列；新增、编辑、重新认证和移除使用串行原子配置事务。
- 首次启动与设置页已复用 `PasswordAccountForm`，设置“账户”分类可管理多个 IMAP/SMTP 密码账户。
- `MailRuntime` 已改为运行期 Supervisor Registry；每个账户独立启动同步、IDLE、退避和待办重放，共享 Repository 及两个主动 IMAP 许可。
- `SendWorker` 已改为账户内 FIFO、账户间轮转、全局两封且每账户一封的调度模型。
- 主窗口已接入动态账户查询、最近账户恢复、账户运行状态、即时本地视图切换和按账户失效事件；最后一个账户移除后不会重新进入数据目录向导。
- 多账户切换菜单已为双行身份信息提供独立行高并保持左对齐；设置页滚动条覆盖在预留槽内，不改变内容宽度；零账户状态会同步清除标题栏侧栏色块。
- Repository 已补强附件、草稿附件、发件任务等账户槽校验，并增加多账户配置补偿、凭据清理、移除影响、FIFO 调度和跨账户隔离回归测试。
- 自动检查在 2026-07-17 执行；结果记录于 `changes/0025-multi-account.md`。完整 Tauri 构建按协作约定未执行，本阶段由用户在开发环境完成手动验收。

## 九、手动验收标准

- 可以在设置中添加至少两个真实 IMAP/SMTP 密码账户；任一协议认证失败时不新增半成品账户。
- 添加成功后无需重启即可出现在主窗口，并开始该账户首次同步。
- 切换账户立即显示各自本地文件夹和邮件；主题、窗口尺寸和应用外观不随账户切换。
- 两个账户可以同时无感接收新邮件；手动收取当前账户不会停止另一个账户的 IDLE/轮询。
- 在两个账户分别执行已读、星标、移动、写信和草稿操作，重启后状态保持且不会串到另一账户。
- 一个账户断网、密码失效或服务器错误时，另一个账户仍可阅读、同步和发送。
- 密码失效账户停止无意义的认证重试，本地邮件仍可读；重新认证成功后无需重启即恢复同步和发件。
- 编辑服务器配置只有在 IMAP/SMTP 都验证成功后生效，失败时旧配置仍可继续工作。
- 有待发送任务或待办操作时不能直接移除账户；允许移除时会停止后台任务、删除本机配置和凭据，但保留匿名本地邮件数据。
- 移除当前账户后界面安全切换到剩余账户；移除最后一个账户后进入正式的“添加账户”状态，但不重新要求选择数据目录。
- `accounts.json` 不包含密码，数据目录不新增邮箱地址、服务器或凭据字段，事件和普通日志不包含秘密或服务器原始响应。
- Windows 10 22H2+ 实机通过；macOS 仅在真实设备或 CI Runner 验证后声明通过。

## 十、明确非目标

- POP3、OAuth、统一收件箱、跨账户全文搜索、账户拖动排序、身份/别名、联系人簿和通知聚合。
- 删除匿名账户槽及其本地邮件、附件和草稿；该破坏性操作留到迁移与硬化阶段统一设计。
- 自动根据邮箱地址把新账户绑定到孤立数据槽；跨机/重新绑定必须使用后续的显式安全流程。
