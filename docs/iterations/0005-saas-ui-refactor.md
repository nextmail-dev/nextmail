# 第五阶段：跨平台窗口壳与 SaaS UI 重构

状态：已完成实现、自动验证与用户手动验收。

日期：2026-07-13

## 阶段定位

本阶段插入在“完整 IMAP 同步语义”和“多账户”之间。它不扩展协议、同步、账户数量或邮件数据模型，只重构窗口系统、视觉基线和现有功能入口，为后续阶段提供稳定桌面壳与设计系统。

## 实施范围

- Windows 主窗口、写信窗口和设置窗口关闭系统装饰，使用自有拖动区域和最小化、最大化、关闭按钮。
- macOS 使用原生 decorations、Overlay 标题栏和原生交通灯；不模拟 macOS 窗口控件。
- 主、写信、设置窗口分别配置窄 Capability；不增加任意 Shell、文件系统或网络权限。
- 拉丁字形与中文字体随应用资源打包，运行时不要求系统预装相同字体。
- 重建设计令牌、浅色/深色中性表面、派生强调色、圆角、阴影、焦点和无边框基础组件。
- 移除主顶部工具栏；左栏沉浸式贯穿窗口，保留拖动宽度和图标折叠；账户菜单承载收取、设置和退出。
- 中栏使用“文件夹标题 + 总数/未读数 + 搜索 + 连续列表”，选中邮件显示强调色左条和柔和背景。
- 阅读器保留发件人和地址文本，星标、回复、回复全部、转发、归档、移动、复制、删除和更多操作改为图标按钮。
- 设置成为单例独立窗口，使用分类侧栏。通用、外观、账户、关于接入现有能力，阅读、写信、通知和高级保留正式空状态。
- 偏好写入成功后由 Rust 发布事件，使主窗口、写信窗口和设置窗口即时同步语言、主题和强调色。

## 非目标

- 不实现多账户新增/删除、POP3、OAuth、模板、签名库、FTS 搜索、通知或托盘。
- 中栏搜索仍只过滤当前已经加载的邮件，不升级为全文索引。
- 不在无 macOS 设备或 Runner 的情况下声明原生交通灯和窗口拖动已通过实机验收。

## 自动验证

- `pnpm test`
- `pnpm build`
- `cargo test --workspace`
- `cargo clippy --workspace --all-targets -- -D warnings`
- 按约定不执行完整 Tauri bundle，保留增量构建缓存和 `dist`。

## 手动验收

- Windows 窗口没有系统标题栏；空白标题区域可拖动，双击可切换最大化，三个窗口按钮行为正确。
- 写信窗口关闭拦截、空草稿丢弃、发送成功后销毁窗口的既有行为未被自绘标题栏破坏。
- 左侧背景贯穿顶部，侧栏和邮件列表仍能拖动，侧栏折叠后所有入口可通过图标与提示访问。
- 顶部工具栏已移除；收取、设置、退出在账户菜单可用；回复/回复全部/转发等操作在阅读器图标区可用。
- 邮件列表的标题、数量、未读数、搜索和选中样式符合参考方向；星标 hover 不出现错误底色。
- 独立设置窗口只能创建一个实例；重复打开会聚焦已有窗口，主题、语言和强调色更改即时反映到其他窗口。
- 中英文、浅色、深色、强调色切换后无原生控件样式、无开发文案、焦点可见且键盘可操作。
- 断开系统字体或在未安装 Roboto/Droid Sans Fallback 的环境仍能正常显示英文和中文界面。
- macOS 12+ 上原生交通灯位置、拖动、最小化、最大化和关闭行为在后续实机或 CI Runner 验证。
