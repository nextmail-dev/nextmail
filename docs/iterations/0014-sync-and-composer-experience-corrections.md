# 第十四阶段：同步与 Composer 体验修正

## 状态

已验收。用户于 2026-07-23 确认本阶段功能与同步调度实机验收通过。

本阶段处理第十三阶段验收后发现的四项明确体验问题，不进入会话、托盘、OAuth、POP3、发布硬化或其他全局优化。

## 范围与结果

- IMAP 继续使用每批最多 100 个 UID 的网络 FETCH，并继续逐封原子落库、逐封发送只含账户/文件夹 ID 与修订信息的失效事件；当前文件夹按事件顺序串行重新读取本地 Query，后一封不会取消前一封刷新。删除了 40ms 前端模拟呈现队列，已有本地首屏仍立即展示。
- 账户 Supervisor 的完整同步入口收敛为首次设定/应用启动后的启动同步、按账户选择的 1/5/10 分钟周期以及手动收取；“仅手动”不启动计时。移除 IDLE、无 IDLE 轮询、固定兜底和秒级失败重试，设置保存及持久化待办不再触发完整同步。
- 账户管理新增自动同步选择并通过数据格式版本 19 按匿名账户槽持久化，默认每 1 分钟。账户真实运行状态进入主工作区 Query 失效，任何实际同步期间手动收信按钮立即禁用并显示旋转状态；Rust 同时拒绝竞态重复请求。
- 设置侧栏当前分类改用主题色派生的淡色背景与主题色文字，不再固定使用白色卡片高亮。
- Composer 工具栏新增图片选择入口；选择图片、直接粘贴图片、富 HTML 中由剪贴板提供字节的图片统一调用既有窄 Command，验证 MIME、魔数和大小后进入内容寻址附件缓存，编辑态使用内存 data URL，持久化仍为 `cid:`。
- 富 HTML 粘贴先通过 Rust 清洗，保留安全结构、class/ID、传统表格属性、行内样式和安全样式表，再把选择器限定到 `data-nextmail-pasted-html` 容器。Tiptap 扩展保留对应样式属性，脚本、事件、表单、嵌入文档、危险 URL、CSS 网络资源和固定遮罩仍被移除。

## 安全与架构边界

- 事件仍只负责本地 Query 失效/重读，不推送邮件列表 DTO 或正文；逐封刷新对应真实原子落库，不降低 IMAP 批取效率，也不在网络等待期间持有 SQLite 写锁。
- 收信按钮状态以账户运行状态为事实来源，Rust 的重复同步保护覆盖前端状态到达前的竞态窗口。
- 未经 Rust 清洗的剪贴板 HTML 不进入 Composer DOM。新增 Command 只接收最多 5 MB HTML 并返回安全片段，不读取文件系统、网络、数据库或凭据。
- 图片仍只接受 PNG、JPEG、GIF、WebP，沿用单项 25 MB、草稿总计 100 MB 限制；远程 `http(s)` 图片不会因粘贴进入 Composer 而被静默下载。
- 不新增依赖、Capability 或 Tauri bundle；新增只保存 `0/1/5/10` 分钟的 SQLite 迁移 0019。

## 手动验收门禁

Windows 10 22H2+ 与 macOS 12+：

1. 使用空本地数据的新账户同步至少 50 封邮件，确认每封落库事件都使当前文件夹连续刷新而不是由前端定时播放；重启已有本地邮件的账户，确认已有首屏立即显示。
2. 在账户管理依次选择“仅手动”、1、5、10 分钟：确认保存设置本身不启动同步；“仅手动”只在点击收取时同步，其他选项只在对应间隔到期时同步，按钮在一次同步期间持续旋转并禁用，完成后稳定恢复，不出现一秒内加载/恢复循环。
3. 新增账户和重启应用各确认只执行一次启动同步；修改正文范围、切换“始终下载非收件箱正文”、已读/星标/移动/删除以及发送/草稿 APPEND 均不得额外触发完整同步。模拟断网后确认不做秒级重复同步，恢复网络后由下一周期或手动收取重试。
4. 在设置窗口切换通用、外观、阅读、写信、通知、高级和关于，确认当前分类始终使用所选主题色的淡色高亮，浅色/深色及不同主题色均正常。
5. 在 Composer 工具栏选择 PNG/JPEG/GIF/WebP，确认图片立即插入光标位置；保存、关闭并重开草稿后图片仍在，实际发件和收件端显示正常。
6. 分别粘贴截图、带图片的富文本，以及含字体、颜色、背景、间距、列表和表格的网页/Office 富文本；确认文字与安全样式基本保持，图片不丢失，仍可编辑并可在 HTML 双栏查看。
7. 粘贴含脚本、事件属性、表单、iframe、固定定位或 CSS 网络资源的 HTML，确认主动内容不执行、不联网、不覆盖应用界面；远程图片仍不会静默加载。
