# 第十二阶段：收信、写信与跨平台体验优化

## 状态

第一至四批已通过 Windows 实机验收（2026-07-22）；第十二阶段 Windows 范围完成。macOS 交通灯、窗口布局与窗口状态仍待 macOS 12+ 实机执行。

本阶段承接 Windows/macOS 实机体验反馈，作为第十一阶段第二批之前的插入迭代。第十一阶段已验收的本地全文搜索保持不变；会话、托盘、通知等未完成范围不在本阶段顺带实施。

## 目标

- 让正文同步更可控、更渐进，并让手动正文下载具有明确进度反馈。
- 让回复/转发草稿只在用户真实编辑后保留，收件人输入具备可验证的标签交互。
- 修复内联 CID 图片、主题高亮、滚动条、分栏拖拽和 macOS 窗口装饰等体验问题。

## 第一批：收信、正文与内联图片

- 新增账户级“始终下载非收件箱邮件正文”设置；默认关闭，不改变既有同步策略。
- 同步时每落库一封邮件即通知本地视图刷新，不再等完整摘要批次结束。
- 手动下载正文显示操作阶段进度；协议库未提供字节流进度时，不伪装成字节百分比。
- 对正文实际引用的安全 CID 图片 part 进行受控内联显示，并从普通附件列表排除；未被正文引用的 part 仍作为附件。

## 第二批：写信与草稿

- 新建收件人标签编辑器；空格、回车及分隔符完成输入，提供格式校验、去重、移除和键盘操作。
- 回复或转发窗口若从未发生用户编辑，关闭时删除本地动作草稿且不排队上传；已编辑或已保存的草稿仍保留。
- 不改变 SMTP、远端 Drafts 同步及模板/签名现有语义。

## 第三批：工作区与跨平台布局

- 文件夹选中态改为主题色淡色背景；邮件列表滚动槽始终可见；账户区不显示同步状态文字。
- 初始向导使用稳定滚动槽，标题栏背景与左右面板连续，不再出现独立空白带。
- 三栏拖拽轨道改为零宽占位、内部覆盖式命中区，消除 1 px 分栏缝隙。
- macOS 移除硬编码交通灯坐标，恢复系统原生定位；仅在 macOS 12+ 实机验证后声明通过。

## 第四批：同步反馈、写信交互与桌面窗口

- 文件夹同步阶段只显示当前文件夹的末级名称，不展示完整父级路径，也不把正文或其他邮件内容放入事件。
- Composer 发件人使用只读地址标签；收件人由空格、回车、逗号或分号即时完成标签，空输入退格时把末尾标签恢复到输入框继续编辑；上方字段使用语义分割线区分。
- 主窗口、设置窗口和写信窗口记忆尺寸与位置；首次没有历史状态时居中。动态 `composer-*` 共享写信窗口类型状态，不按草稿 ID 无限累积窗口记录。
- 删除、归档或移动使邮件离开当前文件夹后，优先选择其下一封，不存在下一封时选择上一封；再次单击已选邮件可清除选择。
- 应用展示性 UI 默认不可选择，输入框、富文本、源码、邮件标题、收发件人和邮件正文等需要复制编辑的内容继续允许选择。
- 邮件列表增加键盘可访问的右键菜单，覆盖回复、回复全部、转发、已读/未读、星标、归档、移动、复制和删除。

本批不改变 IMAP 操作语义、草稿/发件语义、HTML iframe 安全权限或邮件数据结构。

## 数据与安全边界

- 新偏好进入账户同步设置表，不进入系统凭据库；密码和 Token 边界不变。
- CID 只解析当前已下载 MIME 中正文明确引用的安全图片类型，不开放任意文件、任意网络、脚本或 `same-origin` 权限。
- 手动正文下载事件只携带账户、邮件 ID 和阶段进度，不向前端事件推送正文。
- 所有数据访问继续按 `account_slot_id` 隔离，网络等待期间不持有 SQLite 写锁。
