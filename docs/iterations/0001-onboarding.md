# 第一阶段：工程与首次启动闭环

状态：已于 2026-07-12 通过用户手动验收。

## 交付范围

- 移除 Tauri 模板演示功能，建立 React/Rust 模块边界。
- 建立 NextMail 自有设计令牌和基础组件。
- 支持系统、浅色、深色主题，四种强调色以及中文、英文。
- 支持默认或手动选择邮件数据目录，并识别已有 NextMail 数据集。
- 支持内置服务商、DNS SRV、HTTPS autoconfig 和手动服务器配置。
- 支持 IMAP/SMTP 密码或应用专用密码认证，以及无加密、STARTTLS、隐式 TLS。
- 认证成功后将密码写入系统凭据库、账户配置写入系统应用配置区，并进入三栏主界面空状态。

本阶段不包含邮件同步、阅读、写信、模板、签名、OAuth、POP3、托盘或通知。

## 手动验收

1. 在无 NextMail 配置的环境启动应用，确认依次显示欢迎、数据目录、邮件账户三个步骤，且无法绕过首次设置。
2. 在欢迎页确认只能切换语言；主题与强调色不在首次设置或主界面直接显示。
3. 分别测试默认目录、自定义空目录、非空无关目录和不存在的目录。
4. 完成目录初始化，确认可迁移目录不存在邮箱地址、服务器配置或密码。
5. 使用可测试账户自动发现设置；需要时展开并修改手动配置。
6. 确认错误密码、错误端口、证书错误和未确认的无加密连接均无法保存。
7. 使用正确配置完成认证，确认没有收到测试邮件，主界面显示账户已连接和未同步空状态。
8. 重启应用，确认直接进入主界面。
9. 完全退出后移动数据目录，再启动应用，确认显示目录恢复界面而不是创建空目录。
10. 验证中文/英文切换立即生效并在重启后保留；通过持久化偏好验证三种主题和强调色能力仍有效。

## 自动验证

- `pnpm test`
- `pnpm build`
- `cargo test`
- `cargo clippy --all-targets -- -D warnings`
- `pnpm tauri build --debug --no-bundle`

验证结束后保留 `dist`、`src-tauri/target` 等正常构建缓存，以支持增量构建。删除 coverage、临时数据目录、测试凭据、日志、协议探针等可能污染项目或用户环境的测试产物；正式测试代码保留。
