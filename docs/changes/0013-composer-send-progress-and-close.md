# 变更 0013：发送进度弹层与成功关闭修复

日期：2026-07-13

## 问题

- 发件队列和 SMTP 连接状态以内联提示显示在正文编辑区上方，造成编辑区跳动且不符合阻塞发送操作的视觉层级。
- 发送成功后调用普通窗口 `close()`，再次进入关闭前草稿保存拦截；已经进入发件状态的草稿可能使该流程失败，因此写信窗口没有关闭。

## 修复

- 保存最终草稿、加入发件队列和 SMTP 发送期间，显示位于窗口中央的模态加载弹层；背景不可编辑，状态文案通过 ARIA 对话框提供。
- 发送失败仍在写信窗口内显示错误和重试按钮，不关闭窗口。
- 发送成功后直接调用写信窗口已授权的 `destroy()`，跳过关闭前草稿同步流程；关闭事件对已发送状态也采用同一路径。
- 发送中阻止关闭窗口，避免发件状态和写信窗口生命周期出现竞态。

## 验证

- `cargo test --workspace`、`cargo clippy --workspace --all-targets -- -D warnings`、`pnpm test` 与 `pnpm build` 通过。
- 按约定未执行 Tauri 完整构建；窗口关闭行为由 `pnpm tauri dev` 手动验收。
