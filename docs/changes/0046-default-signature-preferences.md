# 0046：默认签名与自动插入偏好

日期：2026-07-22

状态：已于 2026-07-22 通过 Windows 10 22H2+ 手动验收。

## 变更

- 数据格式版本 17 新增 `signature_preferences`，为全局范围和每个匿名账户槽保存单一默认签名、自动插入开关与 revision。
- 迁移从既有四场景签名引用中按新建、回复、回复全部、转发顺序保留一个默认值；没有规则但已有签名的范围选择最早创建项。迁移随后清除场景签名引用，模板规则保持不变。
- 创建某范围第一个签名时，签名记录和默认偏好在同一事务内提交。账户没有显式偏好时继承全局；创建首个账户签名后建立账户默认。
- 新增稳定 `get_signature_preferences` / `save_signature_preferences` Command 和前端 API。签名引用按当前范围验证，写入使用 revision 拒绝陈旧覆盖。
- 写信设置将签名展示为连续列表，提供默认标记、“设为默认”和“自动为邮件选择默认签名”；四场景设置只保留模板选择。
- Composer 首次创建新建、回复、回复全部或转发草稿时读取已解析偏好。关闭自动插入不会影响 Composer 中的手动签名选择，既有草稿也不会被重新插入。

## 边界

- 签名三格式正文仍只保存在 `mail_signatures`，偏好表不复制内容、账户地址、凭据或内部路径。
- React 只使用公开账户 ID 和稳定 DTO；Rust 将其转换为匿名账户槽，SQLx 类型不越过 Repository。
- 本批没有实施账户入口迁移或桌面通知，没有修改 HTML sandbox、SMTP、远端 Drafts 或邮件同步语义。
