# 变更 0021：文件夹树交互、滚动槽与列表日期规则

日期：2026-07-15

- 邮件列表完全隐藏参与布局的 WebView 原生滚动条，改用绝对定位在列表内部的覆盖式自绘细滑块；滑块只在对应区域滚动时显示。
- 邮件列表固定预留 12px 内部空白放置覆盖式滚动条；该空白始终存在，滑块是否显示都不会再次改变邮件行的可用宽度。
- 文件夹展开箭头使用固定的 24px 无背景控制列，叶节点保留等宽占位，父子缩进以 16px 递增，避免按钮背景和间距破坏树形对齐。
- 文件夹栏的覆盖式滑块通过负外边距进入侧栏右侧 padding；“移动到/复制到”菜单的滑块进入弹层右侧 padding。两处列表项继续占用原有主体宽度，不为滑块额外收窄。
- 文件夹父节点不再根据服务端 `selectable` 标记在界面中禁用；点击名称进入该文件夹，点击独立箭头展开或收起子文件夹。
- 文件夹默认全部展开，收起状态只影响当前窗口，不写入账户或邮件数据。
- 邮件列表时间统一按本机日历显示：当天为 `HH:mm`，昨天使用本地化“昨天”，本年度更早邮件为 `MM-dd`，非本年度为 `yyyy-MM-dd`。

## 验证范围

- 文件夹父节点点击、子节点展开/收起与 IMAP 分隔符层级单元测试。
- 当天、昨天、本年度和跨年度日期格式单元测试。
- TypeScript 类型检查与全部前端测试；按约定不执行完整 Tauri 构建。
