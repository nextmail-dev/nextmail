# 0042：收信正文策略、渐进刷新与 CID 图片

日期：2026-07-22

状态：已通过 Windows 实机验收（2026-07-22）。

## 变更

- 新增账户级“始终下载非收件箱邮件正文”，默认关闭；Inbox 继续使用既有 30/90/365 天或全部策略，其他文件夹开启后不设正文时间截止。
- 保留每次最多 100 个 UID 的 IMAP 网络 FETCH，但摘要或正文每成功落库一封就发布邮箱变化，使本地列表逐封出现。
- 手动正文命令新增仅含账户 ID、邮件 ID、阶段和百分比的 `message-body-progress` 事件；前端显示确定型阶段进度条，不把协议库未提供的字节进度伪装成真实下载字节。
- 阅读器按 ADR 0011 将正文实际引用的受限 CID 图片转换为内存 data URL，并从普通附件摘要排除；未引用或不安全的 part 保留为附件。
- 数据格式版本 16 为 `account_sync_settings` 增加非收件箱正文偏好。

## 边界

- 没有新增远程图片代理、任意文件协议、前端 MIME 解析或事件正文传输。
- 网络请求仍批量化，逐封刷新只改变落库后的可见性通知。
