# 变更 0019：桌面工作区细节与发件本机时区

日期：2026-07-15

- 发送成功通知改由页面根节点 Portal 承载，并固定在紧凑标题栏下方；通知不再受三栏工作区 `overflow` 裁剪，也不会被标题栏拖动层遮挡。
- MIME 生成显式写入 RFC 5322 `Date` 头，时间与 UTC 偏移均来自发送时操作系统的本机时区；不再沿用 `mail-builder` 自动生成的 `+0000` 日期。
- 两处分栏从占据 6px 的网格列收窄为 1px；拖动命中区以覆盖方式扩展，hover 或键盘聚焦时显示贯穿整个栏高的细线，连续邮件列表不再出现由拖拽列造成的左右留白。
- 自绘标题栏由 40px 收窄为 34px，Windows 窗口按钮宽度同步收窄；macOS 原生交通灯位置和标题文字避让空间同步缩小，主窗口、写信窗口和设置窗口保持一致。

## 验证范围

- 前端单元测试与 TypeScript 类型检查。
- `nextmail-protocols` MIME 单元测试与 Clippy；测试验证生成的 `Date` 偏移与执行机器本机偏移一致。
- 按项目约定不执行完整 Tauri 构建，Windows/macOS 窗口实际间距和拖拽手感由 `pnpm tauri dev` 实机确认。
