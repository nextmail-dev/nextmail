# 变更 0012：无感同步、可调工作区与基础回复转发

日期：2026-07-13

## 同步体验

- Supervisor 将唤醒原因拆分为待办操作、手动收取和后台同步。已读、未读、星标、移动、复制、删除以及 Sent/Drafts APPEND 只触发待办队列处理，不再额外执行一次完整文件夹同步。
- 首次启动同步和用户点击“收取”时继续更新并展示进度；IMAP IDLE、定时对账和无 IDLE 轮询在后台静默落库，只发送轻量数据变更事件。
- 当前节奏保持不变：支持 IDLE 时新邮件事件会立即唤醒，另有 5 分钟全量对账兜底；不支持 IDLE 时每 60 秒轮询；连接失败从 2 秒指数退避，最大 5 分钟。
- 邮件列表后台失效查询不再显示标题栏 Spinner，新邮件和远端状态变化在查询完成后原位更新。

## 主工作区

- 文件夹栏和邮件列表栏增加键盘可访问的纵向拖动分隔条，文件夹栏限制为 208–360 px，邮件列表限制为 280–560 px，正文至少保留 360 px。
- 文件夹栏可折叠为 64 px 图标模式；账户入口、新建邮件和文件夹仍可操作，图标提供可访问名称和悬停说明。
- 邮件行的悬停背景由整行统一承担，右侧星标区域不再露出不同底色。

## 回复、转发与主工具栏

- 窗口顶部主工具栏增加回复、回复全部、转发和复制到；未选择邮件时自动禁用。
- 回复全部去除当前账户地址并按邮箱地址去重。回复生成 `Re:` 草稿并持久化 `In-Reply-To` 和 `References`，发送与远端草稿 APPEND 均保留线程头。
- 转发生成 `Fwd:` 草稿，带入原邮件正文；原附件会先按需下载，再以内容哈希引用加入转发草稿。
- 第七阶段仍负责回复/转发模板、变量与签名规则，基础动作不再依赖该阶段。

## 数据迁移

- 新增迁移 `0006_reply_forward.sql`，为草稿保存关联邮件、`In-Reply-To` 和 `References`；账户配置和凭据边界不变。

## 验证

- Rust 测试新增回复全部收件人去重、引用链与安全 MIME 线程头覆盖。
- `cargo test --workspace`、`cargo clippy --workspace --all-targets -- -D warnings`、`pnpm test` 和 `pnpm build` 通过。
- 按约定未执行 Tauri 完整构建，桌面行为由 `pnpm tauri dev` 手动验收。
