# 变更 0004：单账户 IMAP 本地阅读核心

日期：2026-07-12

## 变更内容

- 根目录建立 Cargo Workspace，将领域端口、SQLite/内容存储、IMAP/MIME/HTML 安全实现与 Tauri 宿主分包。
- 保留第一阶段数据库迁移校验值，并新增邮件文件夹、规范化邮件、远端位置、正文、附件、同步状态和同步偏好表。
- 增加只读 IMAP 初始/增量同步、有限退避、同步事件、同步范围回填和按需正文/原始 EML 获取。
- 所有本地邮件查询按匿名账户槽校验；密码仍只从系统凭据库读取，不进入数据目录、DTO 或日志。
- 主窗口改为全宽工具栏与三栏布局；工具栏提供设置、账户管理、关于和退出，收取/新建按阶段范围保持禁用。
- 增加文件夹、无限分页邮件列表、安全 HTML/纯文本阅读、附件缓存和原始邮件查看。
- 邮件 HTML 在 Rust 中白名单清洗，再进入无脚本、无 same-origin 的 sandbox iframe；远程图片和导航默认阻止。

## 兼容性

- `.nextmail-data.json` 容器格式仍为版本 1；`content.sqlite` 通过 SQLx 迁移升级到数据格式 2。
- 现有第一阶段数据目录在首次打开邮件 Repository 时自动迁移，不要求重新配置账户。
- 旧的 `src-tauri/migrations` 副本已移除，唯一迁移来源为 `crates/nextmail-storage/migrations`。

## 已知差异

- 远程图片按发件人放行、受控外链、CID 资源协议和下载后系统打开附件尚未启用；详见第二阶段实施文档。
- 后续编码修复批次已补跑 `pnpm test` 与 `pnpm build` 并通过，见变更 0005。
