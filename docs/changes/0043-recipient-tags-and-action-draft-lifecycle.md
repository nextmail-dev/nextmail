# 0043：收件人标签与动作草稿生命周期

日期：2026-07-22

状态：已通过 Windows 实机验收（2026-07-22）。

## 变更

- Composer 的 To/Cc/Bcc 改为标签输入：空格、回车、逗号、分号或失焦提交，支持显示名称、去重、退格删除、按钮删除与内联格式错误。
- 草稿保存前会解析所有尚未提交的输入；无效地址阻止保存、发送或关闭，并给出中英文生产文案。
- 回复/回复全部/转发草稿在创建时标记为“未编辑可丢弃”；任何用户保存会持久化 `user_edited=1`。
- 关闭窗口时先由 Rust 条件删除。被删除的空白或未编辑动作草稿不再排队上传 Drafts；已编辑动作草稿、远端导入草稿和普通非空草稿保持原有保留语义。
- 数据格式版本 16 为草稿增加 `discard_if_untouched` 与 `user_edited`，判断和删除均在账户槽约束的 SQL 中完成。

## 边界

- 没有改变 SMTP envelope、服务器 Drafts 关联头、模板/签名规则或正文编辑模型。
- 前端不能把任意草稿声明为已删除；Rust 返回稳定布尔结果后才决定是否排队远端草稿。
