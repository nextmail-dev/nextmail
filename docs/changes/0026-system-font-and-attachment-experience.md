# 变更 0026：系统字体与附件体验

日期：2026-07-18

## 变更内容

- UI 从随包 Roboto/Droid Sans Fallback 切换为 Windows Segoe UI + 微软雅黑回退、macOS 系统 UI + 苹方回退；保留既有平台字号令牌以便单独比较清晰度。
- 移除两个字体 npm 包、入口 CSS 导入、`@font-face` 和仅供这些字体使用的发布许可证资源。
- 阅读设置新增“下载附件后自动打开”，默认开启；关闭后首次点击只下载，第二次点击仍可手动打开。
- 阅读器附件区域缩小外边距和内边距，并增加顶部间距；已下载附件不再禁用。
- 每个附件新增“另存为”，由 Rust 打开系统保存对话框并写入用户选择的位置。
- 新增安全附件缓存副本和系统打开用例：普通文件交给系统默认程序，高风险扩展名只在文件管理器中显示。

## 安全边界

- 前端只提交账户 ID 和附件 ID，不接收内容哈希、内部附件路径或缓存路径。
- Rust 在下载、打开和另存为之前校验账户数据槽与附件归属，跨账户 ID 返回统一的附件不存在错误。
- 缓存文件名移除路径穿越、控制字符、Windows 非法字符和保留设备名，并限制长度；缓存父目录只使用不透明摘要。
- Tauri opener 不开放给前端 Capability，公共命令不能打开任意路径。高风险附件不直接执行。
