# 0047：账户菜单管理入口

日期：2026-07-22

状态：已于 2026-07-22 通过 Windows 10 22H2+ 手动验收。

## 变更

- 左上角账户身份统一为可打开的账户菜单；即使只有一个账户也保留相同行为，多账户切换项之后通过分割线提供“账户管理”。
- 将既有账户列表、添加表单和账户详情面板提取为主工作区可复用的管理弹窗，保留连接编辑、重新认证、同步策略、非收件箱正文、文件夹映射和安全移除能力。
- 独立设置窗口移除重复的“账户”类别，继续为写信定义保留账户摘要 Query；设置窗口的其他类别和底部设置入口不变。
- 移除当前账户后优先管理另一个现有账户；移除最后一个账户后主窗口展示正式空状态，并由同一管理弹窗继续提供添加入口，不重新进入数据目录向导。
- 新增单账户菜单、管理入口、多账户选择、删除后选择、无账户添加和设置类别收敛的前端回归测试。

## 边界

- 本批复用现有稳定账户 DTO、Command、`src/app/api.ts` 和 TanStack Query key，没有新增账户或凭据协议。
- 密码仍只通过表单交给 Rust 并进入系统凭据库；React 不访问 SQLite、邮件服务器或凭据库。
- 本批不实施通知偏好或通知窗口，不修改首次启动的数据目录语义、IMAP/SMTP 同步语义或平台窗口 Capability。
