# 变更 0020：文件夹树、阅读偏好与界面细节修复

日期：2026-07-15

- 首次启动标题栏与 300px 引导侧栏共用同一宽度令牌；离开主工作区时会清理运行时侧栏宽度，避免标题栏出现错位色块。
- `MailboxSummary` 增加服务端 IMAP 层级分隔符。前端只按该分隔符连接真实父子节点，并显示缩进后的末级名称；不根据 `/` 等字符猜测层级。
- 全局滚动条滑块默认透明，仅在对应容器发生滚动后的短时间内显示，不改变滚动区域宽度。
- 写信窗口移除与系统窗口关闭按钮重复的正文工具栏关闭按钮，并移除“发送”按钮单独的投影。
- 设置“阅读”类别接入“自动加载远程图片”。偏好默认关闭，单独写入系统应用配置区的 `reading-preferences.json`，不会进入可迁移邮件数据目录；Rust 保存后通过窄事件同步所有窗口。
- HTML 清洗继续移除脚本、表单、嵌入文档、事件处理器、外部样式表、危险链接和 CSS 资源 URL，同时允许经过属性白名单过滤的行内排版、尺寸、颜色、表格与间距样式。
- 深色阅读层不再强制清除所有正文子元素的颜色和背景，只提供页面级兜底色，从而保留邮件自己的安全行内样式。
- 数据迁移会把已有 HTML 正文标记为待重建并清除旧清洗缓存；下次后台同步或按需阅读时重新获取正文，纯文本邮件不受影响。

## 保留边界

- 邮件内 `<style>` 样式表仍会移除，因为需要独立 CSS 解析与资源 URL 重写后才能安全放行。
- `cid:`、本地附件和远程图片代理的自定义资源协议仍处于待定状态，本次没有扩大 Tauri 网络或文件系统权限。
- 自动加载远程图片会直接允许当前安全 iframe 请求 HTTP(S) 图片，并保留 `sandbox`、`no-referrer` 和禁止脚本等限制；设置文案明确提示打开跟踪风险。

## 验证范围

- `nextmail-protocols` HTML 安全语料：安全行内排版保留，资源型与越界 CSS 属性移除。
- Tauri 存储单元测试：阅读偏好默认值和原子往返保存。
- 前端文件夹层级、独立设置阅读项、阅读 iframe 与既有组件回归测试。
- TypeScript 类型检查；按约定不执行完整 Tauri 构建，实际滚动条、文件夹缩进与邮件视觉由 `pnpm tauri dev` 实机确认。
