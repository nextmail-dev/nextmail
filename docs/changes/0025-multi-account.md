# 变更 0025：多账户

日期：2026-07-17

## 变更内容

- 将外置账户配置升级为带修订号、最近选择账户和幂等凭据清理队列的多账户集合。
- 首次启动与设置窗口复用同一个账户发现、手动配置、非加密警告和 IMAP/SMTP 双认证表单。
- 新增运行期账户添加、连接编辑、重新认证、移除影响检查、安全移除和账户运行状态命令。
- 将邮件后台重构为按账户管理的 Supervisor Registry，共享 SQLite Repository，并限制主动 IMAP 操作全局最多两个。
- 将发件 Worker 改为账户内 FIFO、账户间轮转，全局最多同时两封且同一账户最多一封。
- 主窗口支持多账户切换、最近选择恢复和账户状态提示；非当前账户事件只失效对应缓存。
- 设置“账户”分类接入账户列表、添加、编辑、重新认证、同步策略、文件夹映射和带影响摘要的二次确认移除。
- 补强附件、草稿附件和发件任务的数据槽校验；账户移除保留匿名数据槽与本地邮件。
- 调整多账户菜单的项目高度与身份左对齐；设置页使用不挤压内容的覆盖式滚动条；移除最后一个账户后清除标题栏的侧栏背景残留。

## 安全与数据影响

- 密码仍只写入 Windows Credential Manager 或 macOS Keychain；编辑表单不会回填密码。
- 更新与移除先把旧凭据清理引用和账户配置原子提交，再执行幂等删除，避免半成功状态。
- 数据目录不新增邮箱、服务器、账户映射或凭据字段；所有邮件数据仍按匿名 `account_slot_id` 隔离。
- 有待发送任务或仍需写回服务器的待办操作时阻止移除；普通本地草稿不阻止。
