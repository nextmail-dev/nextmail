# Windows 中文字体视觉校准

## 背景

随包的中文字体目前只有 400 常规字重，而界面中部分控件和辅助文案使用 11px、13px 字号。它们在 Windows 常见的 125% 或 150% 显示缩放下可能落在半物理像素上；原有全局样式同时禁用了字体字重合成，导致中文强调文字比具有 500/700 字重的拉丁文字更虚。macOS 的 CoreText 表现正常，因此本次不直接替换字体，也不改变 macOS 视觉基线。

## 变更

- React 渲染前检测 Windows、macOS 或其他平台，并写入根节点 `data-platform`；窗口标题栏复用同一平台判断。
- 把原有 11px、13px 界面字号收敛为设计令牌。macOS 和其他平台继续使用 11px、13px，Windows 使用 12px、14px。
- Windows 恢复 WebView/DirectWrite 的平台默认文字平滑和渲染策略，不再强制套用面向 macOS 的抗锯齿参数。
- Windows 允许合成缺失的中文强调字重，使中文与随包 Roboto 的中粗、粗体层级更一致。
- 仅在 Windows 提高浅色和深色主题的辅助文字对比度，减少细灰字的发虚感。
- 当前字体文件、字体许可证和 macOS 字体行为保持不变。

## 验收关注点

- Windows 100%、125%、150% 缩放下，邮件列表、文件夹、菜单和表单辅助文字应比调整前更清晰。
- 中文粗体与拉丁粗体的视觉层级应接近，不应再出现中文标题明显偏细。
- 浅色和深色主题中的次要文字仍保持层级，但不应因灰度过浅而难以辨认。
- macOS 的字号、排版密度和字体渲染不应发生变化。
- 如果 Windows 实机仍有明显锯齿，再单独评估带完整中文字重和 Windows hinting 的宽松许可证字体，避免把字体替换与显示参数调整混在同一轮验证。
