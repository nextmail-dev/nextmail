# 0045：同步反馈、Composer、窗口状态与列表操作

日期：2026-07-22

状态：Windows 10 22H2+ 已验收（2026-07-22）；macOS 窗口状态与原生交通灯位置仍待 macOS 12+ 实机执行。

## 变更

- `SyncProgress` 新增当前文件夹 Unicode 末级显示名；Rust 按 IMAP 服务器声明的层级分隔符从完整名称提取末级名称，启动和手动同步在摘要/正文阶段继续显示“正在同步文件夹 ……”而不是完整路径或泛化阶段文字。事件仍只携带进度元数据。
- Composer 发件人改为只读地址标签，字段间增加语义分割线；收件人只在空格、回车、逗号、分号或失焦时生成标签，空输入退格会把末尾标签恢复到输入框。未提交输入暂停自动保存，避免 800ms 定时器造成延迟标签化，发送/关闭前仍最终校验。
- 引入 Tauri 官方 `tauri-plugin-window-state` 2.4.1，在 Rust 宿主保存尺寸、位置和最大化状态。主窗口、设置窗口与写信窗口首次居中；动态 `composer-*` 共享 `composer` 状态。没有向 React 开放插件命令或 Capability。
- 删除、归档或移动当前邮件后按可见顺序选择下一封，末尾回退上一封；再次单击当前邮件可取消选择。
- 邮件列表新增基于 Radix Context Menu 的右键操作，覆盖回复/回复全部/转发、草稿编辑、已读/未读、星标、归档、移动、复制和删除，全部复用既有稳定 API。
- 全局展示性 UI 禁止网页式文本拖选；表单、富文本、HTML 源码、邮件标题、收发件人、邮件正文和原始邮件源码仍可选择复制。

## 依赖与边界

- 新增官方 `tauri-plugin-window-state`（MIT/Apache-2.0）和与现有组件同体系的 `@radix-ui/react-context-menu`（MIT）。
- 窗口状态位于系统应用配置目录，不进入用户可迁移邮件目录，不含账户、邮件或凭据内容。
- 本批未修改 IMAP 离线操作事务、SMTP/草稿语义、邮件 HTML sandbox 或数据格式版本。
