# 变更 0008：写信窗口、主界面与阅读器修复

日期：2026-07-12

## 写信窗口

- `composer-*` Capability 增加精确的 `core:window:allow-destroy` 权限，关闭事件仍由写信窗口自身处理，不向主窗口开放该权限。
- 关闭时先完成未保存草稿；完全未修改的空草稿通过 Rust 所有权校验后删除，有收件人、主题、正文或附件的草稿继续保留。
- SMTP 成功后立即关闭写信窗口；成功事件由主窗口接收并在右上角显示自动消失的站内通知，不再占用写信表单区域。
- 紧凑字段显式移除原生 input 外观、阴影和边框，修复左侧边线露出。
- 使用 Tiptap 开源 `TextStyleKit` 增加字体、字号、文字颜色和文字背景色；样式继续同时进入 editor JSON 与发件 HTML。

## 主界面

- 参照用户提供的布局层级，将账户身份从全宽工具栏移入左侧栏顶部；工具栏只保留收取、新建、本地草稿、当前文件夹搜索和应用菜单。
- 搜索框位于工具栏右侧，当前阶段对已经加载的当前文件夹邮件按主题、发件人和摘要即时过滤，不提前引入第九阶段 FTS。
- 文件夹和邮件行的 ghost 控件不再保留边框宽度；只在侧栏、工具栏、列表/正文之间保留结构性分隔。
- 深色主题的背景、卡片、弹层、静音色和边框改为无色度灰黑色；强调色仍由用户偏好独立控制。

## 邮件阅读

- Rust 清洗层保留经过协议白名单校验的 `http/https` 图片地址，但默认 iframe CSP 仍只有 `img-src data:`，因此不会产生远程请求。
- 用户点击“立即显示”后，只为当前邮件的 sandbox iframe 放开 `http/https` 图片；仍使用 `no-referrer`，不保存全局偏好。
- 旧缓存中没有保留图片地址时，按钮先重新获取并清洗该邮件正文，再显示远程图片。
- 阅读 iframe 根据 NextMail 的有效主题注入独立配色。深色模式使用灰黑背景，并对常见正文元素增加可读性覆盖；邮件脚本、表单和导航权限保持禁用。

## 验证

- 空草稿删除与非空草稿保留由 SQLite Repository 测试覆盖。
- 远程图片默认 CSP、显式放开和深色正文注入由 Rust/前端测试覆盖。
- Windows debug 应用完成主界面视觉检查；未创建、修改或发送测试邮件。
- `cargo test --workspace` 25 个测试、`pnpm test` 6 个测试、Clippy、格式检查、Vite 生产构建和 Tauri debug 无 bundle 构建均通过。
