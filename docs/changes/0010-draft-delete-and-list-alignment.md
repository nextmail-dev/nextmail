# 变更 0010：草稿安全删除与邮件列表对齐

日期：2026-07-13

## 变更内容

- 本地草稿菜单的每个项目右侧增加迷你删除按钮；第一次点击进入四秒确认状态并切换为危险色垃圾桶图标，第二次点击才执行删除，避免误触。
- 草稿内容与删除按钮是两个并列且独立的 Radix 菜单项，不嵌套交互元素；选择删除按钮不会触发草稿内容项的打开操作。
- 删除操作通过 Rust 业务命令完成，前端不直接改动本地状态；删除成功后只失效当前账户的草稿查询。
- Repository 仅允许删除属于当前匿名账户槽且仍为 `editing` 状态的草稿。已进入发件队列或已经发送的记录不能通过该入口删除。
- 正在独立写信窗口中编辑的草稿禁止从主窗口删除，用户需先关闭写信窗口，避免窗口随后自动保存并重新产生冲突数据。
- 邮件列表标题区域与第一封邮件之间改为零间距；列表行仍使用底部分隔线建立连续边界。
- `Stack` 基础组件新增显式的 `gap="none"` 选项，业务界面不再用局部 class 绕过组件间距约束。

## 验证

- `cargo clippy --workspace --all-targets -- -D warnings`：通过。
- `pnpm test`：通过，共 7 个前端测试；包含删除按钮不会打开草稿的菜单事件回归测试。
- `pnpm build`：通过。
