前端边界：

- React 不直接访问 SQLite、邮件服务器、任意文件系统或系统凭据库。
- 所有 IPC 通过 src/app/api.ts 和稳定 DTO；不要在业务组件散落裸 invoke。
- TanStack Query 管理本地视图；事件只负责失效/刷新，不推完整邮件正文。
- UI 使用 src/components/ui 的自有组件和语义令牌，不使用组件库默认外观或浏览器原生表单样式。
- 业务页面尽量不直接写裸 div/span/input/select；语义与可访问性需要的 HTML 放在基础组件内或有明确理由。
- 中文和英文文案必须同时维护，UI 只能出现生产级文案。
- 保持系统/浅色/深色、主题色、Windows/macOS 平台窗口行为和键盘可访问性。

Rust 与安全边界：

- core 不依赖 Tauri、SQLx 或具体协议库；第三方协议/SQLx 类型不得越过 Adapter/Repository 边界。
- 具体实现由 state.rs 装配并通过 ports 注入；不要在 application/runtime 重新硬编码构造 Adapter。
- Command 保持薄，统一返回稳定 CommandError { code, params, retryable }。
- 密码和 Token 只进入系统凭据库；不得进入 SQLite、可迁移数据目录、前端持久化、日志或错误详情。
- TLS 严格验证系统信任链，不提供忽略证书错误；明文连接必须显式确认。
- 邮件 HTML 继续由 Rust 清洗并在 sandbox iframe 中渲染。不得为显示效果放开 scripts、forms、same-origin、top-navigation、任意文件或任意网络权限。
- 数据访问必须按 account_slot_id 隔离；多表可见写入使用事务；不要持有 SQLite 写锁等待网络或慢文件 I/O。
