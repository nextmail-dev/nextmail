# ADR 0007：多账户运行时与凭据事务

状态：已接受；同步调度部分由 ADR 0013 修订

日期：2026-07-17

## 背景

单账户阶段只有一个同步循环和一组账户配置。多账户要求在不阻塞首帧、不重复打开 SQLite 连接池、不泄漏密码且不让一个服务器故障拖住其他账户的前提下，支持运行期新增、编辑、重新认证和移除。

账户配置与邮件数据还具有不同迁移语义：本机 `account_id`、服务器配置和凭据引用留在系统应用配置区，邮件数据只通过匿名 `data_slot_id` 存在可迁移目录。移除本机账户时默认保留匿名数据槽。

## 决策

- `MailRuntime` 作为 Supervisor Registry，维护 `account_id -> AccountSupervisor`。每个账户独立拥有唤醒、退避、启动同步、待办重放和 IDLE 生命周期，共享一个 `MailRepository`。
- 所有 Repository 操作先把 `account_id` 解析为 `data_slot_id`，再在 SQL 中验证消息、文件夹、附件、草稿、发件任务和待办属于该槽。
- 主动 IMAP 操作共享两个全局许可，同一账户由单一 Supervisor 保持串行；IDLE 等待不占许可。Supervisor 使用单调代次丢弃配置更新或移除后的迟到状态和事件。
- 发件任务继续绑定不可变 MIME 与创建时的数据槽；调度按账户内 FIFO、账户间轮转执行，全局最多两封且每账户最多一封。
- `accounts.json` 使用进程内串行锁、单调修订号和原子文件替换。密码更新先写新凭据，再把新配置和旧凭据清理引用一起提交，最后删除旧凭据。
- 账户移除把配置移除与凭据清理引用一起提交，再执行幂等凭据删除。删除失败不恢复账户；后续启动重试匿名清理引用。

## 影响

- 一个账户的网络、TLS、认证或 SMTP 故障不会停止其他账户的本地阅读、同步和发件。
- 切换账户只改变本地查询范围，不等待网络；非当前账户事件只失效该账户缓存。
- 凭据清理队列可能短暂保留已经成功删除的匿名引用，但重复删除被视为成功，不包含邮箱地址、密码或 Token。
- 移除账户不破坏本地邮件数据；孤立数据槽的识别、重绑定和显式清理由后续迁移与硬化阶段负责。
