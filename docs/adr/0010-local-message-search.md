# ADR 0010：本地邮件全文搜索边界

状态：已接受

日期：2026-07-22

## 背景

主工作区原有搜索只在 React 已加载的分页邮件上匹配主题、摘要和发件人。这既漏掉未加载页面，也无法搜索本地正文、收件人和附件名。第十一阶段需要引入 SQLite FTS5，同时继续遵守匿名账户槽隔离、离线优先、稳定 DTO 和事件仅负责失效查询的既有边界。

中文常见查询可能只有一至两个汉字；SQLite FTS5 trigram 适合连续片段和无空格文本，但不足三个 Unicode 字符时没有可匹配 trigram。直接把用户文本交给 `MATCH` 还会把引号、运算符等解释为查询语法。

## 决策

### 1. 第一批只搜索当前账户的当前文件夹

- 沿用现有“搜索当前文件夹”产品入口，不新增跨账户搜索、统一收件箱或远端服务器搜索。
- `search_messages` 复用 `MessageListPage` 和日期/消息 ID 游标。Rust 先把公开账户 ID 解析为匿名 `account_slot_id`；SQL 同时校验 FTS 行、规范邮件、文件夹和邮件位置的账户/文件夹归属。
- 前端以账户、文件夹和搜索文本组成 TanStack Query key。邮件/文件夹事件沿用 `messages` 前缀失效，不推送搜索结果或正文。

### 2. 索引可搜索表示，不索引 HTML 或秘密

- `message_search` 保存主题、发/收/抄送地址 JSON 的可搜索文本、预览、正文 `plain_text` 和附件名，并保存不参与分词的消息 ID 与匿名账户槽 ID。
- HTML 标记和原始 EML 不进入索引，避免重复语料、标记噪声和把不可信 HTML 当成查询表示。密码、Token、系统凭据引用、内部路径和服务器原始错误从不进入索引。
- FTS 表属于 `content.sqlite` 内的可重建本地视图，会随可迁移数据目录迁移；它不是新的凭据或前端持久化位置。

### 3. 字面片段查询与中文短词降级

- 三个及以上 Unicode 字符使用大小写不敏感、去除变音符的 FTS5 trigram tokenizer。查询作为转义后的单个字面短语绑定，不允许用户输入改变 SQL 或注入 FTS 操作符。
- 一至两个 Unicode 字符在同一索引存储列上执行受账户、文件夹、可见位置和分页约束的字面扫描，以保证常见中文短词可用。该降级不用于更长查询。
- 第一批不做相关度排序、分词器插件、拼写纠正、模糊搜索或结果高亮；结果继续按文件夹中的邮件时间倒序排列。

### 4. 索引与可见邮件写入同步维护

- 数据格式版本 15 的迁移创建 FTS 表、回填既有邮件，并用 SQLite 触发器维护邮件头/预览、正文纯文本和附件名变更。
- 触发器与原有同步/正文写入位于同一 SQLite 事务；不持有写锁等待网络或文件 I/O。删除账户或规范邮件时索引行随消息删除触发器清理。
- 搜索查询只读取本地已经同步且当前可见的邮件位置；正文尚未同步时只能由已存在的头、预览或附件元数据命中，正文到达后自动进入索引。

## 影响

- 搜索覆盖整个当前本地文件夹，不再受 React 已加载页限制，正文、收件人和附件名命中也能返回稳定邮件列表项。
- FTS 表会重复保存一份可搜索文本并增加数据目录体积；换取的是离线搜索和不扫描完整正文的常规查询路径。未来若更换 tokenizer 或语料，只能新增迁移并重建索引，不修改已发布迁移。
- 一至两个字符的降级查询可能比常规 FTS 慢，但范围受当前账户和文件夹约束；后续性能语料表明确认前不引入第三方分词依赖。
- 会话聚合、全局搜索、托盘和通知没有被本决策提前实现。

## 验证门禁

- Rust 回归必须覆盖迁移回填、主题/地址/正文/附件名、中文短词、索引更新、分页、文件夹隔离和账户槽隔离。
- 前端回归必须证明搜索经过防抖 IPC/TanStack Query，且正文或附件命中不会被列表 DTO 二次过滤。
- Windows 使用真实本地数据验证旧数据迁移、新邮件即时入索引、超过一页的结果和多账户/多文件夹隔离；macOS 未执行前不得记录通过。
