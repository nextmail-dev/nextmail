# ADR 0003：持久化 MIME 发件管线

- 状态：接受
- 日期：2026-07-12

## 背景

桌面邮件客户端不能把一次 SMTP 调用等同于“发送”按钮的生命周期。网络中断、窗口关闭或进程崩溃都可能发生在 MIME 已生成但 SMTP 结果尚未确定时；若每次重试重新生成邮件，还可能改变 Message-ID、边界或附件内容。

## 决策

NextMail 在联网前生成完整原始 MIME，原子保存到可迁移数据目录，并在同一业务操作中创建引用该内容哈希的 `send_job`。后台 Worker 只读取不可变 MIME 并使用 SMTP `send_raw` 发送。

- 队列写入成功即代表用户内容已安全落地，不代表远端已经接受。
- 自动重试复用同一 MIME 和 Message-ID。
- `sending` 是可恢复的临时状态；异常启动时回到 `queued`。
- SMTP 成功与 IMAP Sent 归档是两个独立状态。本阶段只完成前者。
- 永久错误保留任务与草稿，由用户修正或显式重试，不删除内容。

## 影响

优点是发送过程可恢复、容易审计且不会依赖写信窗口存活；代价是必须维护任务状态、恢复逻辑和内容文件生命周期。SMTP 在连接断开时可能存在“服务器已接受但客户端未收到确认”的不确定窗口，因此后续重试必须复用 Message-ID，并在未来 Sent 同步中提供去重依据。
