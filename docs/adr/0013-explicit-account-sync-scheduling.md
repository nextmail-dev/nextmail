# ADR 0013：显式账户同步调度

状态：已接受

日期：2026-07-23

## 背景

既有账户 Supervisor 同时接受启动、手动、设置变更、本地待办、Inbox IDLE、无 IDLE 轮询、固定全文件夹兜底和错误退避等唤醒来源。这些来源共享“后台同步请求”标志，普通设置或邮件操作也可能立即触发完整同步；多个通知相邻到达时，界面会出现收信按钮在一秒内反复加载和恢复的循环。

IMAP 每批最多拉取 100 个 UID，但邮件已经逐封原子落库并发送 `mailbox-changed`。前端连续失效默认会取消或合并仍在进行的查询，随后又通过定时呈现队列模拟逐封显示，因此视觉节奏与真实落库事件不一致。

## 决策

- 每个账户的完整同步只有三个运行时入口：Supervisor 创建后的启动同步、账户配置的周期计时到期、用户手动收取。首个账户保存和应用启动都会创建对应 Supervisor，因此自然覆盖首次设定与每次应用启动。
- `account_sync_settings.sync_interval_minutes` 只允许 `0/1/5/10`；`0` 表示仅手动，默认值为 1 分钟。间隔从一次同步完成或设置改变后重新计时。
- 移除 Inbox IDLE、无 IDLE 轮询、固定 5 分钟兜底和秒级完整同步错误退避。同步失败后保持离线或重新认证状态，只等待下一个已配置周期或手动重试。
- Supervisor 分开保存“手动完整同步”和“处理持久化待办”请求。Flags、移动、复制、删除、Sent/Drafts APPEND 与显式待办重试只唤醒待办处理，不触发完整邮箱对账。
- 正文范围与“始终下载非收件箱正文”只持久化，留待下一次合法同步使用；修改同步间隔只唤醒 Supervisor 重新读取设置和重置计时，不立即同步。
- IMAP 网络 FETCH 继续最多 100 UID 一批，单封邮件完成原子落库后立即发出最小 `mailbox-changed`。当前文件夹在 React 侧按事件顺序串行重新读取本地 Query；不推送邮件 DTO 或正文，不使用定时播放队列。

## 影响

- 同步按钮的运行状态只对应一次明确的启动、周期或手动同步，不再被设置保存和本地待办反复触发。
- 选择“仅手动”或较长间隔意味着新邮件不会由服务器推送即时出现；这是显式账户设置的产品取舍。通知与本地视图更新也遵循同一同步时机。
- 网络或服务器短暂失败不会在秒级自动重连；1/5/10 分钟模式会在下一周期再试，仅手动模式需要用户再次收取。
- 连续邮件事件可能产生多次本地只读查询，但保持事件和真实落库顺序；SQLite WAL、账户槽隔离、最多两个主动网络许可和单账户串行边界不变。

## 验证门禁

- Repository 回归覆盖默认 1 分钟、手动/1/5/10 分钟往返和数据格式版本 19。
- DTO 回归固定同步间隔 IPC 值；Supervisor 回归固定四个可选周期语义。
- 前端回归覆盖非当前文件夹失效、当前文件夹串行重读和监听稳定性。
- 实机覆盖首次账户、重启、四个间隔、手动收取、同步失败及大量邮件逐封到达时的按钮状态和列表刷新。
